## 1. 一句话概括

本项目围绕 **diffusion-generated synthetic images 如何真正帮助 few-shot image classification** 展开：先用 Stable Diffusion 生成类别候选图像，再用 CLIP、真实 few-shot 特征、多样性约束和训练阶段策略筛选/使用合成样本，最终在 Flowers20 与 Oxford-IIIT Pets20 上验证哪些选择策略能超过强基线 `clip_topk`。

最新 Pets20 大候选池扩容实验后的当前最高结果是：

```text
sp80 margin_topk + RealFT
= 160 Stable Diffusion candidates/class
+ selected_per_class = 80
+ selection = margin_topk
+ 30 epoch real+synthetic training
+ 5 epoch real-only fine-tuning
```

在 Pets20 5-shot、3 seeds、统一 `30+5 RealFT` 训练日程下达到：

| method / setting | candidates/class | selected/class | accuracy mean | accuracy std | macro F1 mean | macro F1 std | seeds |
|---|---:|---:|---:|---:|---:|---:|---:|
| sp80 margin_topk + RealFT | 160 | 80 | **0.9093** | **0.0023** | **0.9087** | **0.0019** | 3 |
| sp60 ccds + RealFT | 160 | 60 | 0.9073 | 0.0049 | 0.9066 | 0.0047 | 3 |
| sp100 anchored_ccds + RealFT | 160 | 100 | 0.9039 | 0.0028 | 0.9032 | 0.0030 | 3 |
| D7 anchored_ccds + RealFT | 80 | 10 | 0.8999 | 0.0089 | 0.8993 | 0.0090 | 3 |
| clip_topk + RealFT | 80 | 10 | 0.8992 | 0.0102 | 0.8982 | 0.0111 | 3 |
| CFRD top40 w7030 ResNet50 + RealFT | 80 | 10 | 0.8894 | 0.0118 | 0.8881 | 0.0128 | 3 |
| CFRD top40 w7030 DINOv2 + RealFT | 80 | 10 | 0.8876 | 0.0077 | 0.8865 | 0.0068 | 3 |

这轮扩容实验再次修正了此前结论：D7 曾是 Pets20 `80 candidates/class, selected_per_class=10` 下的最高 anchored 配置；但当候选池扩大到 `160 candidates/class`，并允许更多 synthetic images 进入训练后，`selected_per_class=80` 的 `margin_topk` 成为目前最高、最稳定的 3-seed 配置。它相对旧 D7 最优的 accuracy mean 提升约 **+0.94 个百分点**，且 std 从 **0.0089** 降到 **0.0023**。这说明 **margin 策略不是无效，而是在每类只选 10 张时作用发挥不出来；当 synthetic 数量扩大到 60/80/class 后，class-consistency margin 能更好筛除混淆样本并形成稳定收益。**

## 2. 项目基础链路

整个项目的工程链路如下：

```text
few-shot split
→ Stable Diffusion v1.5 按类别生成候选图
→ CLIP target/confuser/margin 打分
→ synthetic sample selection
→ real + selected synthetic 训练 ResNet50 frozen-backbone 分类器
→ 汇总 accuracy / macro F1 / best val
```

核心脚本顺序：

1. `scripts/make_splits.py`：创建 few-shot train/val/test split 和 class map。
2. `scripts/generate_candidates.py`：使用 Stable Diffusion 生成候选图。
3. `scripts/score_candidates.py`：使用 CLIP 计算 target score、confuser score、margin score 和 image embeddings。
4. `scripts/select_candidates.py`：根据不同策略选择每类固定数量的合成样本。
5. `scripts/train_classifier.py`：用 real-only、传统增强或 real+synthetic 训练分类器，并可执行 real-only fine-tuning。
6. `scripts/plot_results.py` 或 `results/research_process/*.csv`：汇总结果和可视化。

当前所有重要产物均按 `project_name` 隔离，例如：

```text
results/<project_name>/generation_metadata.csv
results/<project_name>/clip_scores.csv
results/<project_name>/clip_image_embeddings.npz
results/<project_name>/selected/selected_<strategy>.csv
results/classifier/<project_name>/<method>/seed*/summary.csv
results/research_process/*.csv
```

## 3. 研究问题的逐步收敛

### 3.1 最初问题

最初目标比较直接：

> 用 Stable Diffusion 给 few-shot 分类任务生成额外训练样本，能否比 real-only、traditional augmentation 更好？进一步，能否设计一种 synthetic sample selection 策略，超过简单但很强的 `clip_topk`？

基础假设是：生成图数量很多，但质量参差不齐。选择策略必须同时处理：

- 语义是否正确；
- 是否容易混淆成其他类别；
- 是否过于同质化；
- 是否贴近真实 few-shot 训练分布；
- 是否真正对最终 ResNet50 分类器有帮助。

### 3.2 逐渐形成的核心判断

实验一路下来，形成了一个越来越明确的判断：

```text
CLIP Top-K 很难打败，因为它非常擅长过滤错类和低质量生成图；
但 CLIP Top-K 选择的是“最像文本提示词的图”，不一定是“最能改善分类边界的图”。
```

因此后续创新都围绕这个判断展开：

1. 不要粗暴抛弃 CLIP；
2. 保留 CLIP 的语义过滤能力；
3. 在 CLIP 高质量候选池中引入其他目标：真实分布接近、多样性、类别难度、prototype、替换诊断、训练阶段回拉；
4. 每次转向都用 overlap、score delta、real-sim delta、per-seed accuracy/macro F1 做证据验证。

## 阶段 1：Flowers20 Coupled Prompt 主实验

### 设置

使用 `configs/flowers20_5shot.yaml`：

```text
20 类 × 5-shot
每类 80 张 Stable Diffusion 候选图
每类选择 10 张生成图用于增强
3 seeds × 20 epochs
```

早期 generation prompt 与 CLIP scoring prompt 共用模板，例如：

```text
a high-quality photo of a {class_name} flower
a close-up photo of a {class_name} flower
a realistic photo of a {class_name} flower
a detailed photo of a {class_name} flower in natural light
```

### 关键结果

| method | accuracy mean | macro F1 mean |
|---|---:|---:|
| real_only | 0.8344 | 0.8031 |
| traditional_aug | 0.8269 | 0.7993 |
| diffusion_random | 0.8561 | 0.8206 |
| clip_topk | 0.8614 | 0.8284 |
| margin_topk | 0.8433 | 0.8103 |
| ccds | 0.8508 | 0.8207 |
| anchored_ccds | **0.8667** | **0.8313** |

### 阶段结论

1. 扩散增强整体优于 `real_only` 和 `traditional_aug`。
2. `clip_topk` 是非常强的选择策略。
3. 原始 `ccds` 没有超过 `clip_topk`，说明纯 margin + KMeans diversity 会牺牲部分高置信样本。
4. 新增 `anchored_ccds` 超过 `clip_topk`，说明更稳妥的方向是：

```text
先保留 CLIP Top-K 高质量 anchor，
再用少量 target / margin / diversity reranking 补齐。
```

这也是第一个有效创新点。

## 阶段 2：Prompt Decoupling 消融

### 为什么要做

Coupled prompt 中，Stable Diffusion 生成 prompt 和 CLIP scoring prompt 完全一致。这可能让 `clip_topk` 受益于 prompt coupling：同一套文字既用于生成，又用于筛选。

因此新增 decoupled prompt：

- generation prompt 继续强调生成质量；
- CLIP prompt 改为更中性的类别识别模板。

配置：

```text
configs/flowers20_5shot_clip_decoupled.yaml
project_name: ccds_flowers20_5shot_clip_decoupled
```

CLIP scoring prompt 例如：

```text
a photo of a {class_name} flower
a close-up image of a {class_name} flower
a natural image of a {class_name} flower
a clear photo of the flower species {class_name}
```

### 结果

| prompt setting | method | accuracy mean | accuracy std | macro F1 mean | macro F1 std |
|---|---|---:|---:|---:|---:|
| coupled | anchored_ccds | 0.8667 | 0.0114 | 0.8313 | 0.0186 |
| coupled | clip_topk | 0.8614 | 0.0125 | 0.8284 | 0.0106 |
| decoupled | anchored_ccds | 0.8667 | 0.0085 | 0.8338 | 0.0128 |
| decoupled | clip_topk | **0.8680** | 0.0101 | 0.8333 | 0.0050 |

### 转向原因

Decoupled prompt 让 `clip_topk` 从 0.8614 提升到 0.8680。`anchored_ccds` 在 accuracy 上不再超过 `clip_topk`，但 macro F1 略高、std 更低。

这说明：

```text
clip_topk 的强度不是偶然；
当 CLIP prompt 更适合类别识别时，它会变得更难超越。
```

于是研究重心从“证明 anchored 一定强”转为：

1. 分析 anchored 与 `clip_topk` 的 overlap；
2. 尝试扩大有效差异；
3. 引入真实 few-shot prototype；
4. 判断是否应换数据集验证泛化。

## 阶段 3：Anchored CCDS 参数搜索

### 方法

`anchored_ccds` 的机制是：

```text
每类先保留 anchor_count 张 CLIP Top-K anchor；
从 target-score top_m 候选池中，按质量/边际/多样性贪心补齐到 selected_per_class。
```

典型分数：

```text
score = quality_weight * target_score
      + margin_weight * margin_score
      + diversity_weight * diversity_gain
```

### 搜索方向

在 decoupled prompt 上尝试：

| arm | anchor_count | top_m | quality | margin | diversity | 目的 |
|---|---:|---:|---:|---:|---:|---|
| A2 | 8 | 40 | 0.75 | 0.15 | 0.10 | 更保守 |
| A5 | 8 | 30 | 0.75 | 0.15 | 0.10 | 更严格候选池 |
| A8 | 8 | 40 | 0.85 | 0.05 | 0.10 | 更依赖 target |
| A9 | 8 | 40 | 0.85 | 0.00 | 0.15 | 去掉 margin |

### 结果

正式 3-seed 20epoch：

| setting | accuracy mean | accuracy std | macro F1 mean | macro F1 std |
|---|---:|---:|---:|---:|
| decoupled clip_topk | **0.8680** | 0.0101 | **0.8333** | 0.0050 |
| decoupled anchored baseline / A1 | 0.8667 | 0.0085 | 0.8338 | 0.0128 |
| A8 | 0.8671 | 0.0068 | 0.8332 | 0.0117 |
| A9 | 0.8592 | 0.0116 | 0.8232 | 0.0078 |

### 结论

1. A8 小幅提升稳定性，但仍没有超过 decoupled `clip_topk`。
2. A9 退化明显，说明完全去掉 margin、提高 diversity 并不合适。
3. 继续在线性 target/margin/diversity 权重上微调，收益有限。
4. 更重要的观察是：anchored 与 `clip_topk` 的 selected overlap 很高，替换样本太少，难以产生 1-2 个百分点提升。

因此下一步转向：**让替换更有依据，而不是单纯加大 diversity。**

## 阶段 4：Prototype-aware CCDS 与 Prototype Gate

### 动机

如果仅按文本相似度选择，CLIP Top-K 可能选到“最像类别文本”的图，但不一定贴近当前 few-shot 真实训练样本。因此引入真实 few-shot 图像 prototype：

```text
prototype_score(x) = cosine(CLIP_image(x), mean(CLIP_image(real few-shot images of target class)))
```

Prototype-aware Anchored CCDS 同时考虑：

1. `target_score`：是否像目标类别文本；
2. `margin_score`：是否不容易混淆；
3. `prototype_score`：是否接近该类真实 few-shot 图像原型；
4. `diversity_gain`：是否补充视觉模式。

### P1-P6：线性 prototype 加分

P1-P6 通过降低 anchor、提高 prototype 权重、扩大 top_m 来降低与 `clip_topk` 的 overlap。

| arm | anchor | top_m | target | margin | prototype | diversity | overlap | accuracy | macro F1 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| P1 | 7 | 40 | 0.60 | 0.10 | 0.20 | 0.10 | 94.5% | 0.8570 | 0.8232 |
| P2 | 6 | 40 | 0.60 | 0.10 | 0.20 | 0.10 | 94.5% | 0.8583 | 0.8222 |
| P3 | 7 | 50 | 0.55 | 0.10 | 0.25 | 0.10 | 91.5% | 0.8596 | 0.8248 |
| P4 | 6 | 50 | 0.55 | 0.10 | 0.25 | 0.10 | 91.5% | 0.8570 | 0.8226 |
| P5 | 5 | 50 | 0.50 | 0.10 | 0.30 | 0.10 | 90.0% | 0.8464 | 0.8107 |
| P6 | 5 | 60 | 0.50 | 0.10 | 0.25 | 0.15 | 88.5% | 0.8583 | 0.8236 |

结论：overlap 的确降下来了，但性能没有同步提升。P5 甚至明显退化，说明 prototype 权重过高会牺牲 CLIP 高质量样本。

### G1-G2：Prototype Gate

为避免 prototype 只是线性加分，又尝试 gate：

```text
1. 每类先按 target_score 取 top_m_for_diversity = 60；
2. 保留 anchor_count 张 CLIP Top-K anchor；
3. 非 anchor 候选按 prototype_score 取 prototype_top_m = 30；
4. 在 gate 后候选中用 target + margin + diversity 补齐。
```

结果：

| arm | overlap | accuracy | macro F1 | best val |
|---|---:|---:|---:|---:|
| G1 | 85.0% | 0.8503 | 0.8152 | 0.8500 |
| G2 | 83.0% | 0.8503 | 0.8105 | 0.8550 |

### 这个阶段的价值

虽然 prototype 系列没有形成最终主结果，但它给出很重要的负面经验：

```text
降低 overlap 本身不是目标；
只有替换掉“低价值 CLIP Top-K”并换入“高价值非 Top-K”才可能提升。
```

换句话说，**差异越大不等于方法越好**。这条经验直接影响后续 Pets、CA-CCDS、replacement-aware 和 CFRD 的设计。

## 阶段 6：D2 Anchored 与 overlap 说服力

### 问题

默认 `anchored_ccds` 与 `clip_topk` overlap 高达：

```text
194/200 = 97.0%
```

这容易被质疑：方法是不是只是复制 CLIP Top-K？

### D2 消融

D2 更激进一些：

```text
Default anchored: anchor=7, top_m=40, quality/margin/diversity=0.75/0.15/0.10
D2 anchored:      anchor=6, top_m=60, quality/margin/diversity=0.75/0.10/0.15
```

selection 差异：

| setting | overlap with clip_topk | target score mean | margin score mean |
|---|---:|---:|---:|
| Default anchored | 97.0% | 0.337525 | 0.042159 |
| D2 | 93.0% | 0.337462 | 0.042394 |

3-seed 结果：

| setting | accuracy mean | macro F1 mean | overlap |
|---|---:|---:|---:|
| clip_topk | 0.8809 | 0.8796 | 100% |
| Default anchored | **0.8844** | **0.8831** | 97.0% |
| D2 anchored | 0.8829 | 0.8817 | 93.0% |

### 结论

D2 证明：即使 overlap 从 97% 降到 93%，anchored 仍能超过 `clip_topk`。但 D2 也低于 Default anchored，说明 CLIP Top-K 中确实有很多高价值样本，过多替换会牺牲质量。

这形成了后续设计原则：

```text
应当温和、可解释地替换；
不要为了降低 overlap 而降低 overlap。
```

## 阶段 7：Confusion-Adaptive CCDS / CA-CCDS

### 动机

固定 anchor count 对所有类别一视同仁，但类别难度不同：

- 高 margin 类：CLIP Top-K 更可靠，应保留更多 anchor；
- 低 margin 类：更容易混淆，应允许更多替换。

CA-CCDS 的本质是：

```text
按类别自适应控制与 clip_topk 的 overlap。
```

实现方式：根据每类 CLIP Top-K 的平均 margin 映射 adaptive anchor count。

### CA2 机制诊断

CA2 配置：

```text
min anchor = 6
max anchor = 8
top_m = 50
quality/margin/diversity = 0.70/0.20/0.10
```

诊断：

```text
anchor_counts = {6: 1, 7: 12, 8: 7}
overlap with clip_topk = 187/200 = 93.5%
```

代表类别：

| group | target_class | topk_margin_mean | adaptive_anchor_count | overlap |
|---|---|---:|---:|---:|
| lowest margin | bombay | -0.040508 | 6 | 9/10 |
| high margin | japanese chin | 0.090431 | 8 | 10/10 |
| high margin | keeshond | 0.075321 | 8 | 10/10 |

机制是成立的：困难类确实保留更少 anchor，容易类保留更多 anchor。

### 正式结果

| setting | accuracy mean | macro F1 mean | overlap |
|---|---:|---:|---:|
| clip_topk | 0.8809 | 0.8796 | 100% |
| Default anchored | 0.8844 | 0.8831 | 97.0% |
| D2 anchored | 0.8829 | 0.8817 | 93.0% |
| CA2 / CA-CCDS | 0.8770 | 0.8756 | 93.5% |

### 结论

CA-CCDS 是一个机制更合理、解释性更强的方向，但 CA2 当前结果低于 `clip_topk` 和 fixed anchored。因此它适合作为 negative/diagnostic ablation，而不是主结果。

这个阶段再次强化了一个认识：

```text
更聪明地控制 overlap 仍不够；
选择到的替换样本必须真正改善分类器训练。
```

## 阶段 8：Same-overlap、Replacement-aware 与 weighted augmentation

阶段 8 同时推进三个问题：

1. CA2 的收益/损失是否只是来自 overlap 变化？
2. 能否显式控制每类 non-CLIP-Top-K replacement count？
3. 合成样本是否应该降低 loss weight？

### 8.1 same_overlap_random

构造与 CA2 相同 overlap 的随机替换对照：

```text
overlap = 187/200 = 93.5%
```

作用是验证“有策略替换”是否优于“同 overlap 随机替换”。

### 8.2 replacement-aware CA

CA2 虽然降低 anchor，但后续 rerank 可能又把非-anchor 的 CLIP Top-K 选回来。replacement-aware CA 显式控制 replacement count。

诊断：

| setting | overlap | selected target mean | selected margin mean |
|---|---:|---:|---:|
| same_overlap_random | 93.5% | 0.337036 | 0.041460 |
| replacement-aware CA | 73.0% | 0.337057 | 0.042850 |
| clip_topk reference | 100% | 0.337552 | 0.041714 |

## 阶段 9：CFRD-MMR + Real-only Fine-tuning

### 核心转向

前面的实验说明：继续在 CLIP Top-K 附近做局部替换，提升空间有限。于是新策略改为：

```text
Stable Diffusion candidates
→ CLIP 只做 top-M 语义过滤
→ ResNet50 penultimate real-feature similarity 约束
→ MMR diversity 贪心选择
→ real + synthetic 训练
→ real-only fine-tuning
```

策略名：`cfrd_mmr`，即 **CLIP-Filtered Real-feature Diverse Selection**。

### 方法机制

对每个类别：

1. 按 `target_score` 保留 `cfrd_clip_top_m` 个候选，CLIP 只做语义/质量过滤；
2. 提取真实 few-shot 图和候选图的 ResNet50 penultimate features；
3. 计算候选图到该类真实样本的最大 cosine similarity：

```text
real_sim(x) = max_i cosine(feature(x), feature(real_i))
```

4. 用 MMR 贪心选择：

```text
score(x) = cfrd_real_weight * real_sim(x)
         - cfrd_diversity_weight * max_{s in selected} cosine(feature(x), feature(s))
```

5. 分类训练后增加 `real_finetune_epochs=5`，将决策边界拉回真实 few-shot 分布。

## 阶段 12：RealFT 公平对照与最终结论修正

### 为什么补跑

在阶段 10/11 中，`CFRD-MMR + RealFT` 与 `CFRD-DINOv2 + RealFT` 都采用：

```text
30 epoch real + synthetic training
+ 5 epoch real-only fine-tuning
```

而此前 Pets20 的 `clip_topk`、`anchored_ccds`、D2 等强 baseline 多数是 20 epoch 训练。这样直接对比会产生公平性问题：

```text
CFRD 的提升可能来自 RealFT 和更长训练日程，
不一定来自 CFRD selection 本身。
```

因此补跑公平对照：让 `clip_topk`、Default `anchored_ccds` 和 D2 `anchored_ccds` 也使用同样的 `30+5 RealFT` 日程。

```text
CFRD-MMR 更适合作为真实特征选择方向的探索性/诊断性 ablation，

```

后续优化目标不应再是超过 20epoch `clip_topk = 0.8809`，而应改为超过：

```text
clip_topk + RealFT = 0.8992
```

最优先的后续方向也应相应调整：

1. 围绕 D2 anchored + RealFT 做小范围搜索，因为它已经达到 0.8982，最接近当前最高结果；
2. 对 `clip_topk + RealFT` 做 per-class 分析，找出 D2/CFRD 哪些类别输给它；
3. 如果继续 CFRD，应先解释为什么 real-feature selection 在 RealFT 后不如 CLIP Top-K 稳定，再考虑 fusion/adaptive，而不是直接作为主结果推进。

## 6. 创新思考过程总结

### 6.1 第一层创新：从“选高分图”到“质量 + 多样性”

最初的 `clip_topk` 只关心 target score。原始 CCDS 尝试加入 class-consistency margin 和 embedding diversity：

```text
CLIP target/confuser margin
+ CLIP image embedding diversity
```

但实验说明，直接追求 diversity 会牺牲高置信样本。因此改为 Anchored CCDS：

```text
高质量 anchor 不动，
只在剩余少量 slots 中引入 diversity。
```

这个思路在 Flowers coupled 和 Pets20 上都有效。

### 6.2 第二层创新：从“全局固定替换”到“可解释替换”

当 anchored overlap 过高时，直觉上应降低 overlap。但 prototype/gate、D2、CA 的结果证明：

```text
overlap 降低不是目的；
替换的样本必须比被替换的 CLIP Top-K 样本更有训练价值。
```

因此引入：

- prototype alignment；
- class-adaptive anchor；
- replacement-aware substitution；
- same-overlap random control。

这些方法虽然不是最终最强，但它们把研究从“调参”推进到“解释替换是否有效”。阶段 12 的公平 RealFT 对照进一步说明：在加入强训练策略后，`clip_topk` 本身也能大幅提升，因此自定义选择策略不仅要超过 20epoch CLIP Top-K，还必须超过 `clip_topk + RealFT` 这个更强的新 baseline。

### 6.3 第三层创新：从“CLIP Top-K 附近微调”到“CLIP 只做语义过滤”

前面多轮实验后，真正的转向是 CFRD-MMR：

```text
CLIP 不再决定最终排序，
而是只负责排除明显语义错误/低质量候选；
最终选择由真实 few-shot feature similarity 和 diversity 决定。
```

这解决了 `clip_topk` 的根本局限：它偏向文本典型样本，而不是目标训练分布。

### 6.4 第四层创新：把 selection 与 training stage 联动

CFRD 单独不够，RealFT 才带来显著提升。这说明 synthetic selection 和 training strategy 必须一起设计：

```text
CFRD：给模型提供真实特征附近但更多样的 synthetic support；
RealFT：最后用真实 few-shot 样本校正 synthetic-induced boundary shift。
```

这比单纯筛选更完整，也更容易形成论文/报告里的方法主张。

### 6.5 第五层认识：特征空间要与最终模型对齐

DINOv2-CFRD diagnostics 更好，但最终不如 ResNet50-feature CFRD。由此得到一个很重要的经验：

```text
selection feature 越强不一定越好；
如果最终 classifier 是 ResNet50，那么 ResNet50 feature space 可能更贴近下游决策边界。
```

这为后续工作提供两个方向：

1. 若坚持 DINOv2 selection，应换 DINOv2/ViT classifier 或做 feature fusion；
2. 若最终 classifier 固定为 ResNet50，ResNet50 real-feature selection 反而更合理。

## 7. 当前可主推的贡献点

### 贡献 1：完整 diffusion-augmented few-shot classification pipeline

构建了从 few-shot split、Stable Diffusion 生成、CLIP 打分、多策略选择、ResNet50 分类评估到结果汇总的完整闭环。

可描述为：

```text
A reproducible diffusion-augmented few-shot classification pipeline with candidate generation, CLIP-based scoring, synthetic sample selection, and frozen-backbone classifier evaluation.
```

### 贡献 2：Anchored CCDS

方法：

```text
Preserve high-confidence CLIP Top-K anchors,
then fill remaining slots via quality / margin / diversity reranking.
```

证据：

- Flowers coupled：0.8667 vs `clip_topk` 0.8614；
- Pets20：0.8844 vs `clip_topk` 0.8809；
- D2 ablation：overlap 降到 93% 后仍达到 0.8829，超过 `clip_topk`。

主张：

```text
A small amount of controlled diversity is beneficial, but only when high-confidence CLIP anchors are preserved.
```

### 贡献 3：一组围绕 overlap 的诊断和负面消融

包括：

- prompt decoupling；
- selected overlap diagnostics；
- prototype linear/gated variants；
- class-adaptive anchor；
- same-overlap random；
- replacement-aware substitution。

价值：这些实验解释了为什么“盲目降低 overlap / 提高 diversity”不可行，也为最终 CFRD 转向提供依据。

### 贡献 4：RealFT 公平基线与 CFRD-MMR 诊断

阶段 10 曾提出：

```text
CLIP-Filtered Real-feature Diverse Selection
+ real-only fine-tuning
```

这个方向证明了 RealFT 对 synthetic-to-real boundary shift 的校正很重要，也证明了真实特征选择可以显著改变 selected synthetic distribution。但阶段 12 的公平对照进一步发现：当所有强 baseline 都使用统一 `30+5 RealFT` 日程后，CFRD 不如 `clip_topk + RealFT`。随后阶段 15 的大 synthetic 数量实验进一步刷新最高结果：当前最高不再是 CFRD、D7 或 `clip_topk + RealFT`，而是 `sp80 margin_topk + RealFT`。

| method / setting | accuracy mean | macro F1 mean |
|---|---:|---:|
| clip_topk + RealFT | **0.8992** | **0.8982** |
| D2 anchored + RealFT | 0.8982 | 0.8974 |
| anchored_ccds + RealFT | 0.8957 | 0.8946 |
| CFRD top40 w7030 ResNet50 + RealFT | 0.8894 | 0.8881 |
| CFRD top40 w7030 DINOv2 + RealFT | 0.8876 | 0.8865 |

修正后的主张是：

```text
RealFT is a strong general training strategy;
CLIP Top-K + RealFT is the current strongest fair baseline;
CFRD-MMR is valuable as a real-feature selection diagnostic,
but it is not the final best-performing method under the unified RealFT schedule.
```

### 贡献 5：DINOv2 feature ablation

DINOv2 improves real-sim/redundancy diagnostics but not final ResNet50 classifier accuracy.这个结果支持一个更细的结论：

```text
Feature quality and downstream classifier alignment must both be considered in synthetic sample selection.
```

### 如何解释这轮负面结果

这轮结果不能证明 LoRA 方向无效，但说明当前参数组合不值得直接扩大到全量。可能原因包括：

1. **few-shot LoRA 数据太少**
   只有 100 张训练图，即使 LoRA 只训练 UNet，也可能学不到稳定的数据集分布。

2. **scale=0.7 可能不够强，也可能刚好破坏候选质量**
   当前 LoRA 影响较温和，但结果没有提升；不能排除 scale=1.0 或其他 steps 会更好。

3. **更贴近 train 不等于更有利于分类器**
   LoRA 可能让图像更接近训练图风格，但同时降低候选多样性，导致最终 accuracy 不升反降。

4. **Flowers20 当前 base SD + Anchored 已经很强**
   base SD fair seed0 的 `anchored_ccds` 已达到 0.8848，本身就是较强点。

5. **候选池大小仍需公平控制**
   LoRA 小正式实验是 20 candidates/class；已有 base fair 主要来自原始候选池。虽然最终 selected 都是 10/class、train size 都是 300，但生成候选池大小和分布不是完全等价。

### 当前 LoRA 结论

本轮应记录为工程可行但效果未证实的负面/诊断实验：

```text
Stable Diffusion LoRA train-only domain adaptation pipeline 已经打通；
但 Flowers20 seed0 的 r4 / 100 steps / scale 0.7 / 20 candidates per class 设置未超过 base SD fair baseline；
因此不建议直接扩大到 3 seeds 或 Pets20 全量实验。
```

更合适的下一步不是立刻全量，而是：

1. **先做视觉检查**
   对比 base SD 与 LoRA 生成图，看 LoRA 是否真的更像 Flowers102，是否存在复制训练图、构图过窄或多样性下降。

2. **做分数/特征诊断**
   比较 LoRA 与 base SD 的：

   ```text
   CLIP target_score / margin_score
   selected image diversity
   generated-to-train nearest-neighbor similarity
   ResNet50 real-feature similarity
   per-class selected samples
   ```

3. **只做小网格，不直接全量**
   如果继续 LoRA，优先小范围扫：

   | variant | steps | rank | scale | 目的 |
   |---|---:|---:|---:|---|
   | v1 | 50 | 4 | 0.5 | 更保守，防过拟合 |
   | v2 | 100 | 4 | 1.0 | 提高 LoRA 影响，成本低 |
   | v3 | 300 | 4 | 0.7 | 更强 domain adaptation |
   | v4 | 100 | 8 | 0.7 | 更高容量，但过拟合风险更高 |

如果只选一个下一步，建议先跑 `steps=100, rank=4, scale=1.0`，因为它是在当前负面结果基础上最便宜的改动。

## 9. 可以如何写成论文/报告叙事

一个较自然的叙事结构是：

### 9.1 问题

Few-shot 分类中，扩散模型能生成大量类别图像，但合成图存在错类、同质化、domain gap 和训练噪声。CLIP Top-K 是强基线，但它选择的是最符合文本类别描述的图，不一定最有利于目标分类器。

### 9.2 初始方法

先构建 CCDS：利用 CLIP target/confuser margin 和 embedding diversity 进行选择。但直接追求 diversity 会牺牲高质量样本。

### 9.3 Anchored 改进

提出 Anchored CCDS：保留高置信 CLIP anchor，只替换少量样本以引入 diversity。该方法在 Flowers coupled 和 Pets20 上超过 CLIP Top-K。

### 9.4 诊断与负面结果

通过 prompt decoupling、overlap diagnostics、prototype/gate、CA-CCDS、same-overlap random 等实验发现：

```text
降低 overlap 并不自动提升性能；
过度替换 CLIP Top-K 往往伤害性能；
真正关键是找到对目标分类器有用的 non-Top-K samples。
```

### 9.5 CFRD-MMR 与 RealFT 公平性检查

因此进一步尝试 CFRD-MMR：CLIP 仅做语义过滤，最终选择依据真实 few-shot feature similarity 与 MMR diversity，并在训练后加入 real-only fine-tuning。CFRD-MMR + RealFT 在非公平对比中曾超过 20epoch CLIP Top-K，但这也暴露了一个关键公平性问题：RealFT 和更长训练日程本身可能带来显著提升。

### 9.6 修正后的结论

补跑统一 `30+5 RealFT` 公平对照后，主方法默认参数下，Flowers20 最高是默认 `anchored_ccds + RealFT`，accuracy mean 为 0.8733，macro F1 mean 为 0.8403；Pets20 当时最高是 `clip_topk + RealFT`，accuracy mean 为 0.8992，macro F1 mean 为 0.8982。进一步的 Pets20 anchored 参数搜索中，D7 anchored + RealFT 达到 0.8999 / 0.8993，略高于 `clip_topk + RealFT`。最新阶段 15 将 Pets20 候选池扩大到 160/class，并把 selected_per_class 扩大到 60/80/100 后，`sp80 margin_topk + RealFT` 达到 0.9093 / 0.9087，成为当前最高结果。CFRD-MMR 和 DINOv2-CFRD 补测没有超过它。DINOv2 ablation 仍说明 selection feature 与下游 classifier 的特征空间对齐很重要，但当前项目的最终结论应强调：RealFT 是强通用训练策略，CLIP Top-K + RealFT 是极强 baseline；在大量 synthetic 进入训练时，class-consistency margin 变成最有效选择信号，当前主结果是 `160 candidates/class + selected_per_class=80 + margin_topk + RealFT`。

## 10. 后续工作建议

### 10.1 最优先

1. **围绕 sp80 margin_topk 做稳定性和可解释性分析**
   当前 `160 candidates/class + selected_per_class=80 + margin_topk + RealFT` 为 0.9093 accuracy / 0.9087 macro F1，是 Pets20 当前最高 3-seed 配置。建议优先补 per-class accuracy、selected margin distribution、与 `clip_topk` / D7 / sp60 CCDS 的 selected overlap，解释为什么大量 synthetic 下 margin 信号变得最有效。

2. **确认 selected_per_class sweet spot**
   当前 sp80 最强，sp100 margin_topk 已出现 seed1 明显掉点。建议小范围补 `selected_per_class=70 / 90`，判断 sweet spot 是否稳定在 80 附近，而不是继续盲目增大 synthetic 数量。

3. **补 selection visualization**
   对比 sp80 margin_topk、sp60 CCDS、sp80 clip_topk、D7 anchored 与 CFRD/DINOv2 选图，展示 margin_topk 是否更少选择跨品种混淆图，以及 clip_topk 在 sp80 下为何 seed0 掉到 0.8642。

### 10.2 方法增强

1. **ResNet50 + DINOv2 feature fusion**
   DINOv2 diagnostics 好但 accuracy 略低，可能融合更稳。

2. **DINOv2 classifier / ViT linear probe**
   如果 selection 用 DINOv2，最终 classifier 也用 DINOv2/ViT，测试特征空间对齐假设。

3. **更细的 RealFT schedule 公平 sweep**
   扫 `real_finetune_epochs`、`real_finetune_lr`，但必须对 `clip_topk`、D2、anchored 和 CFRD 同步使用相同 schedule，避免再次出现训练日程不公平。

4. **CFRD per-class adaptive top_m / feature fusion**
   如果继续 CFRD，应以超越 `clip_topk + RealFT` 为目标，尝试 per-class adaptive top_m、ResNet50+DINOv2 feature fusion，或解释 CFRD 在某些类别上的互补价值，而不是直接作为主结果。

5. **LoRA 生成侧诊断，而不是直接全量扩大**
   Stable Diffusion LoRA train-only 链路已经打通，但 Flowers20 seed0 的 `r4/s100/top20/scale0.7` 没有超过 base SD fair baseline。后续如果继续生成侧微调，应先做视觉检查、generated-to-train nearest-neighbor、CLIP score/margin 和 real-feature similarity 诊断；如果要再跑，优先小网格测试 `scale=1.0` 或 `steps=300`，不要直接扩大到 3 seeds / Pets20 全量。

### 10.3 论文风险控制

1. 不要过度声称 “DINOv2 更强所以更好”，当前事实是 diagnostics 更好但 ResNet50 accuracy 略低。
2. 不要把 CA-CCDS 写成主结果，它目前是解释性/负面消融。
3. 不要再把 `CFRD-MMR + RealFT`、D7 或 `clip_topk + RealFT` 写成当前最终最强方法；最新大 synthetic 结果后，当前最高是 `sp80 margin_topk + RealFT = 0.9093`。
4. 所有最终结论必须区分“20epoch baseline 对比”“统一 30+5 RealFT 主方法默认参数公平对照”“Pets20 anchored 调参最优对照”和“Pets20 160 candidates/class 大 synthetic 对照”。当前默认主方法中，Flowers20 最高是 `anchored_ccds + RealFT = 0.8733`，Pets20 默认公平最高曾是 `clip_topk + RealFT = 0.8992`；Pets20 anchored 调参最优中，D7 anchored + RealFT 为 0.8999；最新大 synthetic 对照中，`sp80 margin_topk + RealFT` 为 0.9093，成为当前主结果。
5. 不要把当前 LoRA 写成有效提升结果。Flowers20 seed0 的 LoRA `r4/s100/top20/scale0.7` 只是证明生成侧微调链路可运行，结果为 0.8689，低于 base SD fair seed0 的 `clip_topk=0.8728` 和 `anchored_ccds=0.8848`。

## 11. 推荐最终表述

如果需要简洁描述当前项目成果，可以这样写：

> 本项目构建了一个 diffusion-augmented few-shot image classification pipeline，使用 Stable Diffusion v1.5 生成类别候选图像，利用 CLIP 进行语义一致性和混淆类别打分，并系统比较 random、CLIP Top-K、margin Top-K、CCDS、Anchored CCDS、Prototype/Adaptive variants 与 CFRD-MMR 等选择策略。实验表明，CLIP Top-K 是强基线，盲目提高 diversity 或降低 overlap 往往会损害性能；RealFT（real-only fine-tuning）是非常强的通用训练策略。补跑统一 `30 epoch real+synthetic + 5 epoch real-only fine-tuning` 公平对照后，Flowers20 默认主方法最高为 `anchored_ccds + RealFT`，达到 87.33% accuracy 和 84.03% macro F1；Pets20 默认主方法最高曾为 `clip_topk + RealFT`，达到 89.92% accuracy 和 89.82% macro F1。进一步的 Pets20 anchored 参数搜索中，D7 anchored + RealFT 达到 89.99% accuracy 和 89.93% macro F1。最新将 Pets20 候选池扩大到 160 candidates/class，并把 selected_per_class 扩大到 60/80/100 后，`selected_per_class=80` 的 `margin_topk + RealFT` 达到 90.93% accuracy 和 90.87% macro F1，成为当前最高结果；它比 CLIP zero-shot 高约 5.06 个百分点，比 traditional augmentation 高约 10.45 个百分点，比旧 D7 高约 0.94 个百分点。CFRD-MMR + RealFT 和 DINOv2-CFRD + RealFT 提供了真实特征选择与特征空间对齐的诊断价值，但没有超过当前 sp80 margin_topk 主结果。

## 12. 最短复盘版

如果只保留一条主线：

```text
1. 扩散增强有效，但 synthetic sample selection 是关键。
2. CLIP Top-K 很强，因为它能稳定过滤语义错误和低质量图。
3. 原始 CCDS 直接追求 diversity，会损失高置信样本。
4. Anchored CCDS 通过保留 CLIP anchor + 少量质量/多样性补齐，在 Flowers coupled 和 Pets20 上超过 CLIP Top-K。
5. Prompt decoupling 和 overlap 分析说明：CLIP Top-K 仍是非常强的基线，单纯降低 overlap 没有意义。
6. Prototype、Gate、CA、replacement-aware 等实验提供了重要负面证据：替换必须有效，而不是只要不同。
7. 后续转向 CFRD-MMR：CLIP 只做语义过滤，真实 few-shot feature similarity + MMR diversity 做最终选择；该方向在非公平对比中曾超过 20epoch CLIP Top-K。
8. RealFT 是关键训练策略；补跑公平对照后发现 RealFT 对 CLIP Top-K 同样非常有效。
9. 统一 30+5 RealFT 后，默认主方法中 Flowers20 最高是 `anchored_ccds + RealFT = 0.8733`，Pets20 默认公平最高曾是 `clip_topk + RealFT = 0.8992`；Pets20 D7 anchored 调参变体达到 0.8999，小幅超过 CLIP Top-K + RealFT。
10. 最新 Pets20 160 candidates/class 大 synthetic 实验进一步刷新结果：`selected_per_class=80 + margin_topk + RealFT = 0.9093 accuracy / 0.9087 macro F1`，成为当前最高主结果；它比 CLIP zero-shot 高约 5.06 个百分点，比 traditional augmentation 高约 10.45 个百分点，比旧 D7 高约 0.94 个百分点。
11. MMR / DINOv2 大 synthetic 补测没有超过 sp80 margin_topk：ResNet CFRD sp60/sp80 seed0 为 0.8940/0.8895，DINOv2 CFRD sp60/sp80 seed0 为 0.8637/0.9051；因此当前应作为 ablation，而不是主结果。
12. DINOv2 diagnostics 更好但不如 ResNet50 feature accuracy，说明 selection feature 要与最终 classifier 对齐；后续主攻方向应是围绕 sp80 margin_topk 做 per-class、margin distribution、overlap 和 selected image diagnostics。
13. Stable Diffusion LoRA train-only 微调链路已打通，但 Flowers20 seed0 的 `r4/s100/top20/scale0.7` 没有超过 base SD fair baseline；它目前应作为生成侧负面/诊断实验，而不是主结果。
```
