Training: 30 epochs real+synthetic + 5 epochs real-only fine-tuning (RealFT)

## 3. 关键对比

| setting | selected/class | accuracy mean | macro F1 mean | notes |
|---|---:|---:|---:|---|
| Pets20 fair RealFT clip_topk | 10 | 0.8992091536 | 0.8982194315 | 强 baseline |
| Pets20 D7 anchored + RealFT | 10 | 0.8998822144 | 0.8992820277 | 旧 tuned anchored 强结果 |
| Pets20 160/sp60 CCDS + RealFT | 60 | 0.9072858826 | 0.9066117036 | large synthetic ablation |
| Pets20 160/sp80 margin_topk + RealFT | 80 | **0.9093050648** | **0.9086824557** | 当前最佳 |
| Pets20 160/sp100 margin_topk + RealFT | 100 | 0.8983678277 | 0.8977857397 | sp100 seed1 掉点，说明 synthetic 不是越多越好 |
| Pets20 160/sp100 anchored_ccds + RealFT | 100 | 0.9039205788 | 0.9031948655 | 强但低于 sp80 margin_topk |

解释口径：

1. `clip_topk + RealFT` 是非常强的公平 baseline。
2. 大 synthetic 设置下，class-consistency `margin_score` 成为更有效的选择信号。
3. `selected_per_class=80` 是当前 sweet spot；`sp100 margin_topk` 出现不稳定，说明继续增加 synthetic 可能引入噪声。
4. CFRD-MMR / DINOv2-CFRD / Anchored / CA-CCDS 等保留为重要历史路径和消融诊断，但不是当前最终最强主结果。

## 5. 轻量复查命令

默认只验证已有 artifacts 并重新生成 summary，不重跑重 GPU 流程：

```bash
cd /root/clip_diffusion_fewshot_ccds/.claude/worktrees/anchored-ccds-results
bash scripts/reproduce_best_pets20_margin_topk.sh
```

单独验证：

```bash
PYTHONPATH=src /root/clip_diffusion_fewshot_ccds/.venv/bin/python scripts/verify_best_pets20_artifacts.py --write-report
```

单独重新生成汇总表：

```bash
PYTHONPATH=src /root/clip_diffusion_fewshot_ccds/.venv/bin/python scripts/summarize_best_pets20_results.py
```

重训分类器需要显式开启：

```bash
RUN_HEAVY=1 bash scripts/reproduce_best_pets20_margin_topk.sh
```

注意：`RUN_HEAVY=1` 会跑 seed0/1/2 的 30+5 epoch 分类器训练，属于重 GPU 流程。完整重新生成 Stable Diffusion 候选图和 CLIP 打分更重，不默认执行。
