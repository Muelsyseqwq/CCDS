> 构建一个面向 few-shot 图像分类的扩散生成增强框架：用 Stable Diffusion 生成候选图像，用 CLIP 做类别一致性和混淆类别打分，再通过多种选择策略筛选生成图，最后用冻结 ResNet-50 评估 few-shot 分类效果。

## 2. 核心方法

完整实验链路：

```text
few-shot split
→ Stable Diffusion 生成候选图
→ CLIP target/confuser/margin 打分
→ 候选选择
→ ResNet-50 frozen backbone 分类训练
→ 汇总 accuracy / macro F1
```

已有选择策略包括：

```text
random
clip_topk
margin_topk
ccds
anchored_ccds
prototype_ccds
prototype_gated_ccds
confusion_adaptive_ccds
```

各策略含义：

- `random`：随机选生成图；
- `clip_topk`：按 CLIP target score 选；
- `margin_topk`：按 target score 与最大 confuser score 的 margin 选；
- `ccds`：先按 margin 取候选池，再用 embedding 多样性选择；
- `anchored_ccds`：保留若干 CLIP Top-K 高质量 anchor，再用 target/margin/diversity 补齐；
- `confusion_adaptive_ccds`：按每类 CLIP Top-K 平均 margin 自适应 anchor 数，困难类降低与 `clip_topk` 的 overlap，简单类提高 overlap；
- `prototype_ccds`：在 anchored 的基础上加入真实 few-shot 图像 prototype score；
- `prototype_gated_ccds`：先用 prototype 过滤候选，再做质量/多样性补齐。

## 4. Flowers 阶段已经完成的内容

### 4.1 主实验

使用配置：

```text
configs/flowers20_5shot.yaml
project_name: ccds_flowers20_5shot
```

已完成：

```text
20 类 × 每类 80 张候选图 = 1600 张真实扩散图
4 种基础选择策略
6 种方法 × 3 seeds × 20 epochs 分类训练
```

主结果大致如下：

| method | accuracy mean | macro F1 mean |
|---|---:|---:|
| real_only | 0.8344 | 0.8031 |
| traditional_aug | 0.8269 | 0.7993 |
| diffusion_random | 0.8561 | 0.8206 |
| clip_topk | 0.8614 | 0.8284 |
| margin_topk | 0.8433 | 0.8103 |
| ccds | 0.8508 | 0.8207 |

结论：扩散增强整体优于 real-only 和传统增强；`clip_topk` 是很强 baseline。

### 4.2 Anchored CCDS

`anchored_ccds` 的设计：

```text
每类先保留 7 张 CLIP Top-K anchor；
再用 target + margin + diversity 贪心补齐到 10 张。
```

在 coupled prompt 主实验中：

| method | accuracy mean | macro F1 mean |
|---|---:|---:|
| clip_topk | 0.8614 | 0.8284 |
| anchored_ccds | 0.8667 | 0.8313 |

结论：Anchored CCDS 超过了 CLIP Top-K，说明“保留高质量 anchor + 少量多样性补充”是有效的。

### 4.3 Decoupled Prompt 消融

后来发现 generation prompt 和 CLIP scoring prompt 完全一致可能给 `clip_topk` 带来 coupling 优势，所以做了 decoupled prompt 实验：

```text
configs/flowers20_5shot_clip_decoupled.yaml
project_name: ccds_flowers20_5shot_clip_decoupled
```

decoupled 后：

| setting | method | accuracy mean | macro F1 mean |
|---|---|---:|---:|
| decoupled | clip_topk | 0.8680 | 0.8333 |
| decoupled | anchored_ccds | 0.8667 | 0.8338 |

结论：decoupled prompt 让 CLIP Top-K 更强，Anchored CCDS 接近但没有明显超过。

### 4.4 Prototype CCDS / Gate 搜参

实现了：

```text
src/ccds/prototypes.py
scripts/compute_real_prototypes.py
prototype_ccds
prototype_gated_ccds
```

做了 P1-P6、G1-G2 初筛。

核心结论：

- 线性加 prototype score 有一定效果，但提升有限；
- 降低 overlap 可以做到，但过度替换 CLIP Top-K 会损害性能；

- 因此下一步转向 Oxford-IIIT Pets，验证方法在更细粒度宠物品种分类任务上的泛化能力。

## 5. 当前真正的下一步：CA-CCDS 工程收口与正式验证

Pets 扩展测试已经完成，当前重点从“接入 Pets”转为把 Confusion-Adaptive Anchored CCDS（代码名 `confusion_adaptive_ccds`，下文简称 CA-CCDS）收口为可测试、可 smoke、可诊断、可继续正式验证的一等策略。

CA-CCDS 的核心思想是：固定 anchor 数不能区分类别难度；低 margin 的困难/易混淆类别应保留更少 CLIP Top-K anchor、替换更多样本，高 margin 的简单类别应保留更多高质量 anchor。因此当前 adaptive 实际上是在**按类别自适应控制与 `clip_topk` 的 overlap**。

关键最新结果：在 Oxford-IIIT Pets20 5-shot 上，`anchored_ccds` 已经超过 `clip_topk`：

```text

anchored_ccds - clip_topk = +0.0035 accuracy, +0.0035 macro F1
```

这满足“自定义 CCDS 系列方法比 CLIP Top-K 好一点”的核心目标，但提升幅度较小。

已追加一个扩大差异的 D2 消融：

```text
D2 anchored: anchor=6, top_m=60, quality/margin/diversity=0.75/0.10/0.15

accuracy mean = 0.8829
macro F1 mean = 0.8817
D2 - clip_topk = +0.0020 accuracy, +0.0021 macro F1
```

对比：

```text
Default anchored: overlap 97.0%, accuracy +0.0035 over clip_topk
D2 anchored: overlap 93.0%, accuracy +0.0020 over clip_topk
```

当前最稳妥结论：Default anchored 是性能最优版本，D2 是差异更明显但仍保持优势的 ablation。若继续增强创新性，不建议继续只调固定 anchor/top_m，建议转向 Confusion-Adaptive Anchored CCDS：简单类别保留更多 CLIP Top-K，困难/易混淆类别替换更多样本。
