## 阶段 1：Coupled Prompt 主实验

### 设置

```text

```

## 阶段 2：Prompt Decoupling 消融

### 动机

原始实验中 generation prompt 和 CLIP scoring prompt 完全一致。这样工程上简单，但可能让 `clip_topk` baseline 受益于 prompt coupling：生成图用同一类描述生成，又用同一类描述筛选。

因此新增 prompt decoupling 消融：

- `generation.prompts` 继续关注生成质量。
- `clip.prompts` 改为更偏类别识别的中性模板。

```text

```

```text
a close-up image of a {class_name} flower

```

## 阶段 5：Prototype-aware Anchored CCDS 计划

### 目标

当前用户的明确目标是：

```text
在 decoupled prompt 下，自定义 CCDS 系列方法要比 CLIP Top-K 高 1-2 个百分点。
```

现有结果显示：

- Coupled prompt 下，`anchored_ccds` 已超过 `clip_topk`。
- Decoupled prompt 下，`clip_topk` 增强到 0.8680 Accuracy，`anchored_ccds` / A8 只接近但没有稳定超过。
- Decoupled 下 anchored 与 CLIP Top-K 的 selected overlap 超过 96%，说明当前方法实际替换样本太少，很难带来 1-2 个百分点提升。

因此下一步不再继续 A9 这类“提高 diversity / 去掉 margin”的方向，而是进入 `prototype_ccds`：用真实 few-shot 样本的 CLIP image prototype 约束生成图选择。

### 方法差异

`clip_topk` 只按：

```text
target_score = CLIP(image, target class text)
```

排序选样。

Prototype-aware Anchored CCDS 将同时考虑：

1. `target_score`：生成图是否像目标类别文本；
2. `margin_score`：生成图是否不容易被混淆成其他类别；
3. `prototype_score`：生成图是否接近该类真实 few-shot 图像原型；
4. `diversity_gain`：生成图是否补充了已选样本没有覆盖的视觉模式。

推荐补齐分数：

```text
score(x) = target_weight * z(target_score(x))
         + margin_weight * z(margin_score(x))
         + prototype_weight * z(prototype_score(x))
         + diversity_weight * diversity_gain(x)
```

其中：

```text
prototype_score(x) = cosine(CLIP_image(x), mean(CLIP_image(real few-shot images of target class)))
```

### 与 Pets 扩展的关系

Pets 扩展仍然重要，但应放在 Flowers prototype 阶段之后。原因：

1. 当前代码只支持 Flowers102 split，Pets 需要新增 dataset loader。
2. Pets 是细粒度猫狗品种任务，CLIP Top-K 可能更强；如果 Flowers 上机制还不清楚，直接扩展 Pets 风险较高。
3. Prototype-aware 方法对 Pets 更有意义，因为真实 few-shot breed prototype 可以帮助过滤“像文本但不像真实训练分布”的生成图。

进入 Pets 前置条件：

- `prototype_ccds` 在 Flowers decoupled 下超过 `clip_topk`；或
- 至少证明其在若干混淆类别上有稳定 per-class 改善，并形成可解释结论。

### 2026-06-01 扩大“有效差异”的优化思路

下一轮优化不追求“为了不同而不同”，而是追求“有效差异”：

```text
替换掉 CLIP Top-K 中高分但冗余、容易混淆、或远离真实 few-shot 原型的样本；
换入仍然有足够 target score，但 margin/prototype/diversity 更有利于下游分类器的样本。
```

具体原则：

1. 不能单纯提高 diversity。A9 已说明 `margin=0` 且提高 diversity 会退化。
2. 应主要通过 `prototype_score` 扩大差异，让替换样本更接近真实 few-shot 分布。
3. `anchor_count` 需要从 7/6 进一步降到 5，让每类 10 张中至少 5 张可由 prototype-aware rerank 决定。
4. `top_m` 可从 40 放宽到 50/60，给 prototype 更大的搜索空间，但仍保持质量门槛。
5. overlap 不是越低越好：
   - overlap > 93%：差异仍太小；
   - overlap 85%-92%：较理想；
   - overlap < 80%：可能太激进，要警惕质量下降。

2. 但差异扩大后没有带来预期 1-2 个百分点提升；P3 是当前最高短训 Accuracy，只有 0.8596。
3. P5 说明过度降低 anchor 并提高 prototype 到 0.30 会伤害质量，Accuracy 降到 0.8464。
4. P6 overlap 最低但 Accuracy 只回到 0.8583，说明“差异更大”不自动等于“更有效”。
5. 当前加权式 prototype_ccds 的收益有限，下一步不建议继续盲目调线性权重。

下一步建议：

- 实现 `prototype gate`，把 prototype 从“线性加分项”改为“候选过滤门槛”。
- 推荐 gate 逻辑：每类先取 target top 60，再对非 anchor 候选取 prototype top 25/30，最后用 target + margin + diversity 补齐。

## 阶段 8：Confusion-Adaptive CCDS / CA-CCDS

### 动机

固定 `anchor_count` 的 Anchored CCDS 对每个类别保留相同数量的 CLIP Top-K anchor。Pets20 D2 消融说明：适度降低与 `clip_topk` 的 overlap 仍能保持优势，但简单地全局降低 anchor 会同时影响容易类和困难类。

CA-CCDS 的目标是把 overlap 控制从“全局固定”改成“按类别自适应”：

- 高 margin 类：CLIP Top-K 更可靠，保留更多 anchor，与 `clip_topk` overlap 更高；
- 低 margin 类：类别更易混淆，保留更少 anchor，让 margin/diversity rerank 替换更多样本，与 `clip_topk` overlap 更低。

因此，现在的 adaptive 本质上就是**按照不同类别自适应 overlap**，但实现上通过 adaptive anchor count 间接控制，而不是直接指定每类 overlap 百分比。

### 已接入代码

```text
src/ccds/selection.py
  select_confusion_adaptive_ccds
  _adaptive_anchor_counts

scripts/select_candidates.py
  --strategy confusion_adaptive_ccds
  CA diagnostics CSV

scripts/train_classifier.py
  --method confusion_adaptive_ccds

scripts/run_quick_pipeline.py
  --augmented-methods confusion_adaptive_ccds
```

本轮对 `_adaptive_anchor_counts` 增加了边界约束：

1. `adaptive_anchor_min_count == adaptive_anchor_max_count` 时，所有类使用固定 anchor count；
2. `adaptive_anchor_min_count > adaptive_anchor_max_count` 时抛出错误；
3. 各类别 margin 基本相同时，回退到默认 `anchor_count`，避免没有难度信号时全部压到最小 anchor。

## 阶段 9：Same-overlap / Replacement-aware / Weighted augmentation 创新实验

### 目标

阶段 8 显示 CA2 的机制可解释，但平均性能低于 `clip_topk` / Default anchored / D2。阶段 9 同时推进三个更明确的创新方向：

1. `same_overlap_random`：构造同 overlap 随机替换对照，验证方法收益是否来自“有策略替换”而非“任意降低 overlap”。
2. `replacement_aware_confusion_adaptive_ccds`：显式控制每类 non-CLIP-Top-K replacement count，避免 CA2 降低 anchor 后又把非-anchor 的 CLIP Top-K 样本选回来。

### DINOv2 结论

```text

redundancy_mean:     ResNet50 0.8286 -> DINOv2 0.8149
```

但在当前设置下，DINOv2 不是新的最优主结果：

```text
DINOv2 top40 w7030 RealFT: 0.8876
ResNet50 top40 w7030 RealFT: 0.8894
```

解释：DINOv2 选出来的样本在 DINOv2 真实特征空间里更接近真实且更不冗余，但最终 classifier 仍是 ResNet50 frozen backbone，因此 DINOv2 selection space 与 classifier decision space 存在轻微不对齐。当前主结果仍应保留为：

```text
CFRD-MMR top40 w7030 ResNet50-feature + RealFT
```

DINOv2 可以作为一个重要 ablation：说明“更强/更真实的视觉表征”改善了 selection diagnostics，但不必然提升 ResNet50 classifier 上的最终 accuracy。

后续如果继续 DINOv2，建议不要只继续调 selection，而是测试：

1. DINOv2 selection + DINOv2/ViT classifier linear probe；
2. ResNet50 feature 与 DINOv2 feature 融合的 CFRD；
3. DINOv2 下重新 sweep top_m / real-diversity 权重，而不是直接沿用 ResNet50 最优 top40 w7030。
